# OpenCove Support

Use the right channel so questions, bugs, and security reports do not get mixed together.

## Get Help

- Usage questions, setup issues, and workflow discussions should go to GitHub Discussions.
- Reproducible product bugs should go to GitHub Issues with the provided issue templates.
- Security concerns must not be filed publicly. Use GitHub Security Advisories or the private maintainer contact listed in [SECURITY.md](./SECURITY.md).

## Before Opening an Issue

- Confirm you are on a currently supported OpenCove release.
- Read [README.md](./README.md), [DEVELOPMENT.md](./DEVELOPMENT.md), and [CONTRIBUTING.md](./CONTRIBUTING.md) if the problem is related to setup or local development.
- Include platform, OpenCove version, reproduction steps, and screenshots or logs when they materially help.

## Scope

- We can usually help with OpenCove bugs, packaging issues, documentation gaps, and contribution workflow questions.
- We may not be able to debug third-party CLI providers, local shell customizations, or machine-specific environment problems without a minimal reproduction.
